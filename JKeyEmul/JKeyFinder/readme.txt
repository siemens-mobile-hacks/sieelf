Если вы читаете эту чушь, значит вам зачем-то понадобились исходники JKeyEmul (хрень какая-то ...)

Итак, как работать с утилитой JKeyFinder?


(Способ #1 (Замороченный))

1) Запускаем.
2) Длиное нажатие на '*' - активация.
3) Нажимаем на клавишу которая будет впоследствии иметь далее введёный код (Например, на правый софт).
4) Теперь вводим сам код клавиши в формате DEC:

 а) Нажимаем на одну из цифр (0-9).
 б) Ещё раз Нажимаем на одну из цифр (0-9).

Теперь у нас есть 2-х значное число. 
(Например, сначала вводим '0', потом вводим '7', получилось '07') //1
(Например, сначала вводим '7', потом вводим '1', получилось '71') //2

5) Длинное нажатие на '#' - сохранение результата с выводом сообщения,
которое показывает код реальной клавиши и код клавиши, которая она будет эмулировать:
///
Реал. кн.: 04 // Это означает, что нажимаем на Правый софт, а она выдаст 07, т.е в Java (для Sony Ericsson) произойдёт действие,
Эмул. кн.: 07 // например, открылось меню - интуитивно догадываемся, что это Правый софт Sony Ericsson, а может и ошибаемся!

Ну если уверены, то потом вводим в конфиге JKeyEmul -> Алт. клава -> Правый софт: 07,
теперь в Java-приложении при JKeyEmul (c включённой Эмуляции ALT клавы) в демонах и при нажатии на правый софт Java воспримет,
как правый софт Sony Ericsson выскочет, например, меню.
///
Если код нажатой клавиши из пункта (3) совпадает с введёным кодом клавиши, 
то возникает ошибка и перекидывает на пункт (3).

6) Ещё Длинное нажатие на '*' сброс всего, переход к пункту (2), Или длинное нажатие на '#' переход к пункту (4).


(Способ #2 (Простейший))
1) Запускаем.
2) Длиное нажатие на '#' - активация.
3) Короткое нажатие на '6' - код клавиши плюс +1 (Начинаем с 0 до 99)
   Короткое нажатие на '4' - код клавиши минус -1
   Короткое нажатие на '5' - эмуляция клавиши с полученным кодом.
   Короткое нажатие на '0' - вывод сообщения с текущим кодом клавиши.
4) Короткое нажатие на '#' - сброс настроек, переход на пункт (2).

PS: Короче, чушь это всё !!!
